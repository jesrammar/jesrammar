## Proyecto destacado

### [spring-dicebear-proxy-cache](https://github.com/jesrammar/spring-dicebear-proxy-cache)

Servicio backend en **Spring Boot** que actúa como **proxy frente a una API externa**, reduciendo llamadas repetidas mediante **caché**, añadiendo **resiliencia ante fallos** y exponiendo **documentación y observabilidad**.

**Qué problema resuelve**

- Evita depender de forma directa del proveedor externo en cada llamada.
- Reduce peticiones repetidas y mejora el comportamiento ante latencia, errores o rate limiting.
- Añade una capa backend controlada, documentada y preparada para evolucionar.

**Stack usado**

`Java 21` · `Spring Boot 3` · `WebClient` · `Caffeine` · `Resilience4j` · `OpenAPI / Swagger` · `Spring Actuator` · `JUnit 5` · `WireMock` · `GitHub Actions`

**Qué demuestra técnicamente**

- Diseño de una API backend con integración hacia terceros.
- Uso de caché para mejorar eficiencia y desacoplar consumo externo.
- Resiliencia aplicada a integraciones con servicios remotos.
- Testing de comportamiento e integración controlada con mocks.
- Documentación y observabilidad como parte del propio servicio.

**Quickstart**

```bash
mvn spring-boot:run
```

Si mantienes la configuración por defecto, la documentación y los endpoints operativos suelen quedar accesibles en:

- `Swagger UI`: `http://localhost:8080/swagger-ui/index.html`
- `OpenAPI`: `http://localhost:8080/v3/api-docs`
- `Actuator`: `http://localhost:8080/actuator`
